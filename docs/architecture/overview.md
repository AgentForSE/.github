# 一、产品定位与分工

AgentForSE 面向**装备软件 / 嵌入式软件**研发场景，以多智能体协同（MetaGPT 理念）覆盖
"需求 → 安全设计 → 编码 → 测试 → 审查 → 交付"全生命周期。当前已立项的四大智能体软件
分工如下：

| 软件 | 定位 | 核心方法 | 研发生命周期阶段 |
|------|------|----------|------------------|
| **SafeWise**    | 系统安全分析 | STPA 三阶段多 Agent 流水线              | 需求 / 系统安全分析阶段 |
| **SafeMatrix**  | 软件功能安全分析 | SFTA + FMEA 两阶段流水线                | 需求 / 设计安全分析阶段 |
| **SmartInsight**| 智能代码走查 | LangChain ReAct Agentic 搜索            | 编码实现 / 代码审查阶段 |
| **SmartST**     | 系统测试自动化 | 需求驱动测试全链路                       | 测试验证阶段 |

> 注：SmartUT（单元测试）作为 SmartST 的层次互补，目前处于早期研发与原型阶段，
> 共享同一套底座规范；本套架构文档为其预留接入点，但当前阶段不在重点建设之列。

## 设计理念（MetaGPT 落地）

- **双引擎驱动**：以大模型为推理引擎，以规则引擎为质量门禁，确保结果合规。
- **Agent 流水线协作**：多智能体并行替代人工串行流程。
- **数据结构化与可追溯**：文档 → 结构化数据 → 自动输出，研发资产沉淀复用。
- **最终目标**：构建"装备软件工厂"，像现代化产线一样高效产出高质量软件。

## 各产品里程碑能力（截至 2026-04）

> 摘自《功能规格及进展统计.xlsx》，便于把握共性需求来源。

- **SafeWise**：已具备 STPA 三阶段（危险与需求 / 控制结构与 UCA / 致因与测试）
  多 Agent 流水线、WebSocket 实时进度、损失→危险→约束→需求→UCA→测试用例全链路追溯、
  OpenAI 生态多模型兼容（DeepSeek / GPT-4o / Qwen / Claude）。
- **SafeMatrix**：已具备 SFTA + FMEA 一体化深度分析、故障树可视化画布、
  项目版本管理与 Diff、多 Agent Prompt 配置、Word 报告（含 PNG + RPN 色标）。
- **SmartInsight**：已具备 LangChain ReAct Agentic 自主搜索、12+ 静态分析报告归一化、
  误报 AI 智能确认、MySQL 经验库、RepoMap 仓结构地图、PPT/Word 报告导出。
- **SmartST**：已具备文档解析与需求结构化、需求确认状态、AI 用例草稿生成、
  增量重解析、定向重生成、执行回填、覆盖率统计与 Excel 导出。

这些已实现能力中的**共性部分**，构成本架构所抽取的[公共底座](./platform-modules/)。
